## 2. 托管架构验收（必须）

- [ ] K1. 生产私钥不以明文形式落盘、入仓库、入镜像。  
- [ ] K2. 私钥托管在 KMS/HSM（或经批准的等效方案）。  
- [ ] K3. 业务仅调用“签名接口”，不能直接导出私钥原文。  
- [ ] K4. 原生链 `ed25519` 与 EVM `secp256k1` 密钥空间分离管理。

## 3. 访问控制与审批（必须）

- [ ] A1. 最小权限 IAM/RBAC 已配置（签名、查看、轮换权限分离）。

- [ ] A3. 生产凭据访问全量审计可追踪。  
- [ ] A4. 已验证越权访问会被拒绝并记录审计日志。

## 4. 轮换流程验收（必须）

- [ ] R1. 有明确轮换周期（建议 <= 90 天）。  
- [ ] R2. 支持双 key 并行窗口（新签名 + 旧验签）。  
- [ ] R3. 支持紧急吊销（泄露时分钟级禁用）。  
- [ ] R4. 轮换 Runbook 已文档化并完成演练。  
- [ ] R5. 轮换后旧 key 在窗口结束后确实失效。

## 7. 必跑演练用例（验收脚本）

### Case-1 正常签名（双算法）
- [ ] ed25519 签名成功 + 验签成功  
- [ ] secp256k1 签名成功 + 验签成功

### Case-2 权限拒绝
- [ ] 未授权主体调用签名接口被拒绝  
- [ ] 审计日志记录完整

### Case-3 轮换窗口
- [ ] 新 key 生效签名  
- [ ] 旧 key 仍可验签（窗口期）

### Case-4 窗口结束
- [ ] 旧 key 被禁用  
- [ ] 旧 key 新签名请求失败并告警

### Case-5 紧急吊销
- [ ] 人工触发吊销流程  
- [ ] 业务恢复完成并有复盘记录
