# Engineering Redlines（禁止自作聪明）

目标：避免“看起来可用、实际掩盖故障”的实现进入主干。

## 核心红线

1. `fail-fast` 是默认策略  
   - 配置非法、存储后端不可用、协议载荷非法，必须返回错误并中断流程。  
   - 禁止自动回退到 `default`、`mem`、隐式兼容路径。

2. 禁止静默吞错  
   - 关键路径禁止“记录 warning 后继续正常运行”。  
   - 错误必须可观测（error 返回、panic、或明确失败状态）。

3. 兼容开关必须显式  
   - 兼容逻辑必须有明确开关，默认关闭。  
   - 未评审通过不得开启旧路径。

4. 每个修复必须有反向测试  
   - 至少新增一个测试证明：旧行为会掩盖问题，新行为会显式失败。

## 自动门禁

- 本仓库使用 `scripts/no_silent_fallback.sh` 做红线静态检查。
- 脚本支持 `-d/--dir` 指定一个或多个目录；不传时默认自动发现并检查当前工作区下的项目。
- CI 工作流：`.github/workflows/redline-guard.yml`。
- 发布门禁：`scripts/run_tests.sh` 已将 redline guard 设为阻断步骤。

## 例外机制（谨慎使用）

- 如必须保留例外，在代码行尾添加 `REDLINE_ALLOW`，并附原因注释。  
- 例外会被审查；无理由注释视为违规。
